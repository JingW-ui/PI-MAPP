树（Tree）是计算机科学中一种非常重要的**非线性数据结构**，广泛应用于文件系统、数据库索引、编译器语法分析、人工智能决策等领域。下面是对树的基础知识的简要介绍：

---

### 一、基本概念

- **节点（Node）**：树的基本组成单元。每个节点可以包含数据和指向其他节点的链接。
- **根节点（Root）**：树的最顶层节点，没有父节点。
- **父节点（Parent）** 和 **子节点（Child）**：若一个节点 A 指向另一个节点 B，则 A 是 B 的父节点，B 是 A 的子节点。
- **叶子节点（Leaf）**：没有子节点的节点。
- **边（Edge）**：连接两个节点的线，表示父子关系。
- **路径（Path）**：从一个节点到另一个节点所经过的节点序列。
- **深度（Depth）**：从根节点到该节点的路径长度（根节点深度为 0 或 1，视定义而定）。
- **高度（Height）**：从该节点到最远叶子节点的最长路径长度；整棵树的高度是根节点的高度。
- **层级（Level）**：通常根为第 1 层或第 0 层，往下依次递增。

---

### 二、树的性质

- 一棵有 $n$ 个节点的树，有 **$n - 1$** 条边。
- 树中**任意两个节点之间有且仅有一条路径**。
- 树是**无环的连通图**。

---

### 三、常见树的类型

1. **普通树（General Tree）**  
   每个节点可以有任意数量的子节点。

2. **二叉树（Binary Tree）**  
   每个节点最多有两个子节点，分别称为左孩子和右孩子。
   - **满二叉树（Full Binary Tree）**：每个节点要么是叶子，要么有两个子节点。
   - **完全二叉树（Complete Binary Tree）**：除最后一层外，其他层都被填满，且最后一层节点靠左排列。
   - **完美二叉树（Perfect Binary Tree）**：所有内部节点都有两个子节点，且所有叶子在同一层。

3. **二叉搜索树（BST, Binary Search Tree）**  
   对于任意节点，其左子树的所有节点值小于它，右子树的所有节点值大于它（常用于高效查找）。

4. **平衡二叉树（如 AVL 树、红黑树）**  
   通过旋转等操作保持左右子树高度差不超过一定范围，以保证操作效率。

5. **多叉树（如 B 树、B+ 树）**  
   常用于数据库和文件系统，支持大量数据的高效磁盘读写。

6. **堆（Heap）**  
   一种特殊的完全二叉树，满足堆性质（最大堆/最小堆），常用于优先队列。

---

### 四、树的遍历方式

对树的访问顺序主要有以下几种：

- **深度优先遍历（DFS）**
  - 前序遍历（根 → 左 → 右）
  - 中序遍历（左 → 根 → 右）——对 BST 可得到有序序列
  - 后序遍历（左 → 右 → 根）

- **广度优先遍历（BFS）** / 层序遍历  
  按层级从上到下、从左到右访问节点，通常用队列实现。

---

### 五、应用场景举例

- 文件系统的目录结构（树形结构）
- HTML DOM 树
- 决策树（机器学习）
- 表达式解析（语法树）
- 路由表（Trie 树）
- 数据库索引（B+ 树）
